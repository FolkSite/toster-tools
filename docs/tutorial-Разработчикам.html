Разработчикам

Подготовка к работе

$ git clone git@github.com:yarkovaleksei/{{EXT_REPO}}.git
$ cd {{EXT_REPO}}
$ npm install

В проекте используется система сборки Gulp, по этому вместо npm run можно напрямую обращаться к сборщику gulp, вызывая его в терминале/коммандной строке, из корня папки проекта.

Проверка кода

Проверка исходного кода Babel-JS в папке ./src/_js/

$ gulp eslint

Сборка

Производится компиляция LESS в CSS, Babel-JS в JS, копирование дополнительных файлов в папку temp. Которую так же можно использовать для загрузки распакованного расширения, с последующей отладкой в браузере.

$ gulp compile

Упаковка в zip перед публикацией

Пакует содержимое ./temp/ в архив, и помещает его в папку ./build/.

$ gulp zip

!!! Перед вызовом этой команды вызывайте 

$ gulp compile

Подписывание расширения для Firefox

Чтобы иметь возможность подписать файл для Firefox надо пройти по этой ссылке, и сохранить ваши ключи в секцию FFSign файла vars.json в папке репозитория. После чего переключить FFSign.use в True, дабы команда создания релиза подхватила вызов создания расширения для FireFox.

Где USERID это "JWT issuer", а SECRETKEY это "JWT secret".

Секция FFSign должна выглядеть так:

"FFSign":{
    "use":true,
    "USERID":"user:123456789:999",
    "SECRETKEY":"e1373f71d00aad90183b410c2e8661265a71845fd0805bd2bdca5fe66a635cfb"
}

Команда для создания и подписи расширения для FireFox:

$ gulp ffsign

Сборка релиза

Производится компиляция LESS в CSS, Babel-JS в JS, копирование дополнительных файлов, упаковка расширения в ZIP архив, подпись расширения для FF, перемещение его в папку build, генерация документации.

$ gulp release

Сгенерировать документацию

$ gulp jsdoc

Документация будет сгенерирована в папку docs в корне проекта.
